## Objetivo

Demonstrar domínio de configurações avançadas do Nginx como proxy reverso e load balancer, implementando alta disponibilidade e distribuição de carga.

## Tarefa

Criar um sistema de e-commerce com múltiplas instâncias de backend, load balancer inteligente e configurações avançadas de proxy reverso.

## Especificações

### Sistema E-commerce
Desenvolver um sistema com:
- **Load Balancer:** Nginx distribuindo carga
- **Frontend:** Loja virtual estática
- **Backend API:** 3 instâncias para alta disponibilidade
- **Admin Panel:** Interface de administração
- **Monitoramento:** Endpoint de status e métricas

### Requisitos Técnicos

**1. Load Balancer Avançado:**
- Algoritmo least_conn para distribuição
- Health checks personalizados
- Failover automático
- Logs detalhados com upstream info
- Rate limiting para proteção

**2. Múltiplas Instâncias Backend:**
- 3 instâncias da API rodando
- Cada instância com ID único
- Simulação de diferentes cargas de trabalho
- Endpoint de health check personalizado

**3. Configurações Avançadas:**
- Virtual hosts para diferentes domínios
- Proxy headers corretos
- Timeout configurations
- SSL termination (certificado self-signed)
- Gzip compression

**4. Monitoramento:**
- Endpoint /status com métricas
- Logs estruturados
- Dashboard simples de status

## 🔧 Nginx Configuration Exemplo

```nginx
# nginx.conf
events {
    worker_connections 1024;
}

http {
    # Log format com informações de upstream
    log_format upstream_log '$remote_addr - $remote_user [$time_local] '
                           '"$request" $status $body_bytes_sent '
                           '"$http_referer" "$http_user_agent" '
                           'upstream_addr: $upstream_addr '
                           'upstream_response_time: $upstream_response_time '
                           'upstream_status: $upstream_status';

    # Rate limiting
    limit_req_zone $binary_remote_addr zone=api:10m rate=10r/s;
    limit_req_zone $binary_remote_addr zone=general:10m rate=1r/s;

    # Upstream para API backend
    upstream api_backend {
        least_conn;
        server backend1:5000 max_fails=3 fail_timeout=30s;
        server backend2:5000 max_fails=3 fail_timeout=30s;
        server backend3:5000 max_fails=3 fail_timeout=30s;
    }

    # Gzip compression
    gzip on;
    gzip_vary on;
    gzip_min_length 1024;
    gzip_types text/plain text/css application/json application/javascript;

    # SSL configuration
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-RSA-AES256-GCM-SHA512:DHE-RSA-AES256-GCM-SHA512;
    ssl_prefer_server_ciphers off;

    # Main server block
    server {
        listen 80;
        listen 443 ssl http2;
        server_name localhost ecommerce.local;

        # SSL certificates
        ssl_certificate /etc/nginx/ssl/cert.pem;
        ssl_certificate_key /etc/nginx/ssl/key.pem;

        # Access log with upstream info
        access_log /var/log/nginx/access.log upstream_log;

        # Frontend (static files)
        location / {
            root /usr/share/nginx/html;
            index index.html;
            try_files $uri $uri/ =404;
        }

        # API proxy with load balancing
        location /api/ {
            limit_req zone=api burst=20 nodelay;
            
            proxy_pass http://api_backend;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
            
            # Timeouts
            proxy_connect_timeout 30s;
            proxy_send_timeout 30s;
            proxy_read_timeout 30s;
            
            # Health check
            proxy_next_upstream error timeout invalid_header http_500 http_502 http_503;
        }

        # Admin panel
        location /admin/ {
            limit_req zone=general burst=5 nodelay;
            proxy_pass http://admin:8080/;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        # Status endpoint
        location /nginx-status {
            stub_status on;
            access_log off;
            allow 127.0.0.1;
            allow 172.16.0.0/12;
            deny all;
        }

        # Health check endpoint
        location /health {
            access_log off;
            return 200 "healthy\n";
            add_header Content-Type text/plain;
        }
    }
}
```

## Docker Compose Exemplo

```yaml
version: '3.8'

services:
  nginx:
    image: nginx:alpine
    container_name: ecommerce-nginx
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - ./nginx/ssl:/etc/nginx/ssl:ro
      - ./frontend/dist:/usr/share/nginx/html:ro
      - nginx-logs:/var/log/nginx
    depends_on:
      - backend1
      - backend2
      - backend3
      - admin
    networks:
      - ecommerce-network
    restart: unless-stopped

  backend1:
    build: ./backend
    container_name: ecommerce-backend1
    environment:
      - INSTANCE_ID=backend1
      - PORT=5000
    expose:
      - "5000"
    networks:
      - ecommerce-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  backend2:
    build: ./backend
    container_name: ecommerce-backend2
    environment:
      - INSTANCE_ID=backend2
      - PORT=5000
    expose:
      - "5000"
    networks:
      - ecommerce-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  backend3:
    build: ./backend
    container_name: ecommerce-backend3
    environment:
      - INSTANCE_ID=backend3
      - PORT=5000
    expose:
      - "5000"
    networks:
      - ecommerce-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  admin:
    build: ./admin
    container_name: ecommerce-admin
    expose:
      - "8080"
    networks:
      - ecommerce-network
    restart: unless-stopped

networks:
  ecommerce-network:
    driver: bridge

volumes:
  nginx-logs:
    driver: local
```
