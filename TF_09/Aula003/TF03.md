# TF03 - Orquestração Multi-Serviços com Docker Compose
**Disciplina:** Implementação de Software  
**Curso:** Análise e Desenvolvimento de Sistemas - UniFAAT  
**Aula:** 03 - Orquestração Local e Persistência  
**Valor:** 2 pontos  
**Prazo:** 1 semana após a aula

## Objetivo

Demonstrar domínio de orquestração de múltiplos serviços usando Docker Compose, implementando comunicação inter-container e persistência de dados.

## Tarefa

Criar um sistema completo de blog com frontend, backend API e banco de dados, todos orquestrados via Docker Compose.

## Especificações

### Sistema de Blog
Desenvolver um sistema com:
- **Frontend:** Interface web para visualizar posts
- **Backend:** API REST para gerenciar posts
- **Banco de Dados:** MySQL/PostgreSQL para persistência
- **Proxy Reverso:** Nginx como load balancer

### Requisitos Técnicos

**1. Arquitetura Multi-Container:**
- **nginx-proxy:** Load balancer na porta 80
- **frontend:** Interface web (porta 3000 interna)
- **backend:** API REST (porta 5000 interna)
- **database:** MySQL/PostgreSQL com volume persistente

**2. Comunicação Inter-Container:**
- Frontend consome API do backend
- Backend conecta no banco de dados
- Nginx faz proxy reverso para frontend e API
- Rede personalizada para isolamento

**3. Persistência de Dados:**
- Volume nomeado para banco de dados
- Volume para logs do Nginx
- Dados devem sobreviver a restart dos containers

**4. Funcionalidades:**
- Listar posts existentes
- Visualizar post individual
- Formulário para criar novo post
- API com endpoints REST completos

## Docker Compose Exemplo

```yaml
version: '3.8'

services:
  nginx-proxy:
    image: nginx:alpine
    container_name: blog-proxy
    ports:
      - "80:80"
    volumes:
      - ./nginx/nginx.conf:/etc/nginx/nginx.conf:ro
      - nginx-logs:/var/log/nginx
    depends_on:
      - frontend
      - backend
    networks:
      - blog-network
    restart: unless-stopped

  frontend:
    build: ./frontend
    container_name: blog-frontend
    expose:
      - "3000"
    depends_on:
      - backend
    networks:
      - blog-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:3000"]
      interval: 30s
      timeout: 10s
      retries: 3

  backend:
    build: ./backend
    container_name: blog-backend
    expose:
      - "5000"
    environment:
      - DB_HOST=database
      - DB_USER=bloguser
      - DB_PASSWORD=blogpass
      - DB_NAME=blogdb
    depends_on:
      database:
        condition: service_healthy
    networks:
      - blog-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:5000/health"]
      interval: 30s
      timeout: 10s
      retries: 3

  database:
    image: mysql:8.0
    container_name: blog-database
    environment:
      - MYSQL_ROOT_PASSWORD=rootpass
      - MYSQL_DATABASE=blogdb
      - MYSQL_USER=bloguser
      - MYSQL_PASSWORD=blogpass
    volumes:
      - blog-data:/var/lib/mysql
      - ./database/init.sql:/docker-entrypoint-initdb.d/init.sql:ro
    networks:
      - blog-network
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "mysqladmin", "ping", "-h", "localhost"]
      interval: 10s
      timeout: 5s
      retries: 5

networks:
  blog-network:
    driver: bridge

volumes:
  blog-data:
    driver: local
  nginx-logs:
    driver: local
```
