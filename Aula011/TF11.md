# TF11 - Sistema de Portfólio com S3 e CloudFront

**Disciplina:** Implementação de Sistemas  
**Curso:** Análise e Desenvolvimento de Sistemas - UniFAAT  
**Valor:** 1.5 pontos  
**Tipo:** Trabalho prático individual  
**Prazo:** Conforme cronograma da disciplina

## Objetivo

Desenvolver um sistema completo de portfólio pessoal utilizando Amazon S3 para hospedagem estática e armazenamento de assets, integrado com CloudFront CDN, demonstrando domínio de armazenamento em nuvem e distribuição de conteúdo.

## Descrição da Atividade

Você deve criar um portfólio pessoal profissional que demonstre suas habilidades técnicas, projetos desenvolvidos e experiência, utilizando as melhores práticas de hospedagem estática na AWS.

## Cenário do Projeto

### Sistema de Portfólio Profissional
Desenvolva um portfólio que contenha:
- **Página Principal**: Apresentação pessoal e resumo
- **Projetos**: Galeria de projetos com detalhes técnicos
- **Experiência**: Timeline de experiências profissionais/acadêmicas
- **Contato**: Formulário de contato funcional
- **Blog**: Seção de artigos técnicos (opcional)

### Arquitetura Proposta
```
Usuário → CloudFront → S3 Static Website
                    ↓
                S3 Assets Bucket → Lambda (processamento)
                    ↓
                API Gateway → DynamoDB (contatos)
```

## Especificações Técnicas

### Frontend Requirements
- **HTML5**: Semântico e acessível
- **CSS3**: Responsivo, moderno (Grid/Flexbox)
- **JavaScript**: ES6+, sem frameworks obrigatórios
- **Performance**: PageSpeed > 90, GTmetrix > A
- **SEO**: Meta tags, structured data

### AWS Services
- **S3**: Static website hosting, lifecycle policies
- **CloudFront**: Global CDN, HTTPS, compression
- **Lambda**: Image processing, form handling
- **API Gateway**: RESTful APIs
- **DynamoDB**: NoSQL database (opcional)

### Security Requirements
- **HTTPS**: Obrigatório em todas as páginas
- **Headers**: Security headers configurados
- **Policies**: Bucket policies restritivas
- **Access**: Princípio do menor privilégio

## Dicas para Sucesso

### Design e UX
1. **Mantenha simples** - Foque na usabilidade
2. **Seja profissional** - Este é seu cartão de visitas
3. **Otimize imagens** - Use WebP, lazy loading
4. **Teste em dispositivos** - Responsividade é crucial

### Performance
1. **Minifique assets** - CSS, JS, HTML
2. **Use CDN** - CloudFront para tudo
3. **Cache inteligente** - Configure TTLs adequados
4. **Monitore métricas** - PageSpeed, Core Web Vitals

### Segurança
1. **HTTPS sempre** - Nunca HTTP em produção
2. **Políticas restritivas** - Menor privilégio
3. **Headers de segurança** - CSP, HSTS, etc.
4. **Monitore acessos** - CloudTrail, access logs

### Custos
1. **Use Free Tier** - Aproveite limites gratuitos
2. **Monitore gastos** - Billing alerts
3. **Otimize storage** - Lifecycle policies
4. **Limpe recursos** - Delete o que não usar
