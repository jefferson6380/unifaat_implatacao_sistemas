## 1. Conceitos Fundamentais

### Amazon S3: Simple Storage Service

**Amazon S3** é um serviço de armazenamento de objetos que oferece escalabilidade, disponibilidade de dados, segurança e performance líderes do setor. É como ter um "drive infinito" na nuvem onde você pode armazenar qualquer tipo de arquivo.

#### Analogia: S3 como Biblioteca Digital Infinita
- **Buckets** = Estantes temáticas (cada uma com nome único)
- **Objects** = Livros/documentos (arquivos de qualquer tipo)
- **Keys** = Código de catalogação (caminho único do arquivo)
- **Metadata** = Ficha catalográfica (informações sobre o arquivo)

### Conceitos Básicos do S3

#### Buckets (Baldes)
**Buckets** são contêineres para objetos no S3. Cada bucket:
- Tem nome globalmente único
- Pertence a uma região específica
- Pode conter número ilimitado de objetos
- Pode ter políticas de acesso específicas

#### Objects (Objetos)
**Objects** são os arquivos armazenados no S3:
- Tamanho: 0 bytes até 5TB por objeto
- Compostos por: dados + metadata + key
- Versionamento opcional
- Podem ter diferentes classes de armazenamento

#### Keys (Chaves)
**Keys** são identificadores únicos dos objetos:
- Funciona como caminho do arquivo
- Exemplo: `images/profile/user123.jpg`
- Case-sensitive
- Pode simular estrutura de pastas

### Classes de Armazenamento S3

#### 1. **S3 Standard**
**Uso**: Dados acessados frequentemente

**Características:**
- **Durabilidade**: 99.999999999% (11 noves)
- **Disponibilidade**: 99.99%
- **Latência**: Milissegundos
- **Custo**: Mais alto, mas sem taxas de recuperação

**Quando usar:**
- Sites e aplicações web
- Distribuição de conteúdo
- Analytics em tempo real
- Aplicações móveis

#### 2. **S3 Intelligent-Tiering**
**Uso**: Dados com padrões de acesso desconhecidos ou variáveis

**Como funciona:**
- Monitora padrões de acesso automaticamente
- Move objetos entre tiers de acesso frequente e infrequente
- Sem taxas de recuperação ou impacto na performance

**Vantagens:**
- Otimização automática de custos
- Sem overhead operacional
- Ideal para dados com padrões imprevisíveis

#### 3. **S3 Standard-IA (Infrequent Access)**
**Uso**: Dados acessados com menos frequência, mas que precisam de acesso rápido

**Características:**
- **Custo de armazenamento**: ~50% menor que Standard
- **Taxa de recuperação**: Por GB recuperado
- **Durabilidade**: 99.999999999% (11 noves)
- **Disponibilidade**: 99.9%

**Casos de uso:**
- Backup de longo prazo
- Disaster recovery
- Arquivos antigos mas importantes

#### 4. **S3 One Zone-IA**
**Uso**: Dados infrequentemente acessados que não requerem múltiplas AZs

**Características:**
- **Custo**: 20% menor que Standard-IA
- **Disponibilidade**: 99.5% (menor que outras classes)
- **Armazenamento**: Apenas uma AZ
- **Risco**: Perda de dados se AZ falhar

**Quando usar:**
- Dados que podem ser recriados
- Cópias secundárias de backup
- Dados não críticos

#### 5. **S3 Glacier Instant Retrieval**
**Uso**: Arquivamento com acesso instantâneo raro

**Características:**
- **Custo**: ~68% menor que Standard-IA
- **Recuperação**: Milissegundos
- **Acesso mínimo**: 90 dias
- **Ideal para**: Dados acessados uma vez por trimestre

#### 6. **S3 Glacier Flexible Retrieval**
**Uso**: Arquivamento de longo prazo com acesso ocasional

**Características:**
- **Custo**: Muito baixo para armazenamento
- **Recuperação**: 1-12 horas
- **Opções de recuperação**:
  - Expedited: 1-5 minutos (mais caro)
  - Standard: 3-5 horas
  - Bulk: 5-12 horas (mais barato)

#### 7. **S3 Glacier Deep Archive**
**Uso**: Arquivamento de muito longo prazo

**Características:**
- **Custo**: Menor custo de armazenamento
- **Recuperação**: 12-48 horas
- **Retenção mínima**: 180 dias
- **Ideal para**: Compliance, backup de 7-10 anos

### Static Website Hosting

#### O que é Static Website Hosting?
**Static Website Hosting** permite hospedar sites estáticos diretamente no S3, sem necessidade de servidores web tradicionais.

#### Vantagens dos Sites Estáticos:
- **Performance**: Carregamento muito rápido
- **Escalabilidade**: Suporta milhões de usuários
- **Custo**: Muito baixo
- **Segurança**: Menos vetores de ataque
- **Simplicidade**: Fácil de manter

#### Limitações:
- Apenas conteúdo estático (HTML, CSS, JS)
- Sem processamento server-side
- Sem banco de dados direto
- Funcionalidades dinâmicas via JavaScript/APIs

#### Configuração Básica:
1. **Criar bucket** com nome do domínio
2. **Habilitar static website hosting**
3. **Definir index document** (ex: index.html)
4. **Configurar error document** (ex: error.html)
5. **Configurar permissões públicas**

#### Exemplo de Estrutura:
```
meusite.com/
├── index.html          # Página principal
├── error.html          # Página de erro
├── css/
│   └── style.css
├── js/
│   └── app.js
└── images/
    └── logo.png
```

### Segurança no S3

#### Bucket Policies
**Bucket Policies** são documentos JSON que definem permissões no nível do bucket:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::meu-site/*"
    }
  ]
}
```

#### Access Control Lists (ACLs)
**ACLs** fornecem controle de acesso no nível do objeto:
- **Private**: Apenas o owner
- **Public-read**: Leitura pública
- **Public-read-write**: Leitura e escrita pública (não recomendado)

#### IAM Policies
**IAM Policies** controlam acesso programático:
- Anexadas a usuários, grupos ou roles
- Mais granulares que bucket policies
- Podem combinar múltiplos serviços AWS

#### Block Public Access
**Configuração de segurança** que previne acesso público acidental:
- Block public ACLs
- Ignore public ACLs
- Block public bucket policies
- Restrict public buckets

### CloudFront: Content Delivery Network

#### O que é CloudFront?
**CloudFront** é o CDN da AWS que distribui conteúdo globalmente com baixa latência.

#### Como Funciona:
1. **Usuário** solicita conteúdo
2. **CloudFront** verifica edge location mais próxima
3. **Se cached**: Retorna conteúdo imediatamente
4. **Se não cached**: Busca no origin (S3) e cacheia
5. **Próximas requisições**: Servidas do cache

#### Benefícios:
- **Performance**: Latência reduzida
- **Escalabilidade**: Suporta picos de tráfego
- **Segurança**: Proteção DDoS, SSL/TLS
- **Custo**: Reduz custos de transferência

#### Configuração Típica:
- **Origin**: Bucket S3
- **Distribution**: Configuração do CDN
- **Behaviors**: Regras de cache
- **Invalidations**: Limpeza de cache

### Versionamento e Lifecycle

#### Versionamento
**Versionamento** mantém múltiplas versões do mesmo objeto:
- **Habilitado**: Novas versões são criadas
- **Suspenso**: Para de criar versões
- **Nunca habilitado**: Estado padrão

**Benefícios:**
- Proteção contra exclusão acidental
- Histórico de mudanças
- Rollback fácil

**Custos:**
- Cada versão é cobrada separadamente
- Pode aumentar custos significativamente

#### Lifecycle Policies
**Lifecycle Policies** automatizam transições entre classes de armazenamento:

```json
{
  "Rules": [
    {
      "Status": "Enabled",
      "Transitions": [
        {
          "Days": 30,
          "StorageClass": "STANDARD_IA"
        },
        {
          "Days": 90,
          "StorageClass": "GLACIER"
        }
      ]
    }
  ]
}
```

### Casos de Uso Práticos

#### 1. **Website Estático**
- **Frontend**: React, Vue, Angular
- **Hospedagem**: S3 Static Website

#### 2. **Backup e Arquivamento**
- **Dados críticos**: Standard ou Standard-IA

#### 3. **Distribuição de Conteúdo**
- **Imagens**: S3 + CloudFront

#### 4. **Data Lake**
- **Raw data**: S3 Standard
- **Processed data**: S3 IA
- **Analytics**: Athena, EMR
- **Archive**: Glacier classes

### Integração com Aplicações

#### Upload Direto do Browser
```javascript
// Configuração do AWS SDK
AWS.config.update({
  accessKeyId: 'YOUR_ACCESS_KEY',
  secretAccessKey: 'YOUR_SECRET_KEY',
  region: 'us-east-1'
});

const s3 = new AWS.S3();

// Upload de arquivo
const uploadFile = (file) => {
  const params = {
    Bucket: 'meu-bucket',
    Key: `uploads/${file.name}`,
    Body: file,
    ContentType: file.type
  };
  
  return s3.upload(params).promise();
};
```

#### Signed URLs
**Signed URLs** permitem acesso temporário a objetos privados:
```javascript
const getSignedUrl = (key) => {
  const params = {
    Bucket: 'meu-bucket-privado',
    Key: key,
    Expires: 3600 // 1 hora
  };
  
  return s3.getSignedUrl('getObject', params);
};
```

#### Pre-signed POST
**Pre-signed POST** permite upload direto do browser para S3:
```javascript
const createPresignedPost = (key) => {
  const params = {
    Bucket: 'meu-bucket',
    Fields: {
      key: key
    },
    Expires: 3600,
    Conditions: [
      ['content-length-range', 0, 10485760] // Max 10MB
    ]
  };
  
  return s3.createPresignedPost(params);
};
```

### Monitoramento e Logs

#### CloudWatch Metrics
- **BucketSizeBytes**: Tamanho do bucket
- **NumberOfObjects**: Número de objetos
- **AllRequests**: Total de requisições
- **GetRequests**: Requisições GET
- **PutRequests**: Requisições PUT

#### Access Logs
**Server Access Logs** registram requisições ao bucket:
- Quem acessou
- Quando acessou
- Qual objeto
- Resposta HTTP
- User agent

#### CloudTrail
**CloudTrail** registra chamadas de API:
- Criação/exclusão de buckets
- Mudanças de políticas
- Configurações de lifecycle
- Ações administrativas

### Otimização de Custos

#### Estratégias:
1. **Escolha da classe correta** baseada no padrão de acesso
2. **Lifecycle policies** para transições automáticas
3. **Intelligent Tiering** para padrões desconhecidos
4. **Compressão** de arquivos antes do upload
5. **Limpeza regular** de versões antigas
6. **Monitoramento** de custos com CloudWatch

#### Calculadora de Custos:
- **Storage**: Classe × GB × mês
- **Requests**: Tipo × quantidade
- **Data Transfer**: GB transferido
- **Management**: Lifecycle, analytics, etc.

## 2. Questões de Múltipla Escolha

### Questão 1
Qual classe de armazenamento S3 é mais adequada para backup de longo prazo com acesso raro?

a) S3 Standard
b) S3 Standard-IA
c) S3 Glacier Deep Archive
d) S3 Intelligent-Tiering

**Gabarito: C**

**Justificativa:** S3 Glacier Deep Archive é projetado especificamente para arquivamento de longo prazo com acesso muito raro (uma ou duas vezes por ano). Oferece o menor custo de armazenamento, mas com tempo de recuperação de 12-48 horas, sendo ideal para backup de compliance e retenção de longo prazo.

### Questão 2
O que é necessário para hospedar um site estático no S3?

a) Instância EC2 para processar HTML
b) Bucket público com static website hosting habilitado
c) Load balancer para distribuir tráfego
d) Banco de dados para armazenar conteúdo

**Gabarito: B**

**Justificativa:** Para hospedar um site estático no S3, é necessário um bucket configurado com static website hosting habilitado e permissões públicas de leitura. O S3 serve os arquivos HTML, CSS e JavaScript diretamente, sem necessidade de servidores, load balancers ou bancos de dados.

### Questão 3
Qual é a durabilidade oferecida pelo Amazon S3?

a) 99.9% (3 noves)
b) 99.99% (4 noves)
c) 99.999999999% (11 noves)
d) 100% (sem perda de dados)

**Gabarito: C**

**Justificativa:** Amazon S3 oferece 99.999999999% (11 noves) de durabilidade, o que significa que se você armazenar 10 milhões de objetos, pode esperar perder apenas 1 objeto a cada 10.000 anos. Isso é alcançado através da replicação automática dos dados em múltiplas instalações.
