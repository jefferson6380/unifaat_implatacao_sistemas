## Cenário: Publicando uma Imagem Docker no ECR
Vamos simular a fase de CI (Continuous Integration), onde a imagem construída localmente é armazenada de forma segura no repositório gerenciado da AWS.

## Seção 1: Preparação da Imagem Local e Definição de Variáveis
### 1.1 Criando o Diretório de Trabalho

```bash
# Criar diretório principal
mkdir -p ~/aulas_lab/aula012
cd ~/aulas_lab/aula012
```
### 1.2 Criando Arquivos Simulados

Vamos criar os arquivos necessários para nosso projeto Python simulado:

```bash
# Inicialize o VSCode
code .
```

### 1.2 Criação do Dockerfile Base

**Escrevendo o Dockerfile Inicial**

```Dockerfile
FROM nginx:alpine

COPY . /usr/share/nginx/html

EXPOSE 80

CMD ["nginx", "-g", "daemon off;"]
```

1.  **Criar/Verificar Imagem Local:** Utilize a imagem criada 
```bash
IMAGE_TAG="V1.0"
# Constrói uma imagem Docker com o nome 'web-app-v1' usando o Dockerfile no diretório atual
docker build -t web-app-v1:$IMAGE_TAG .

# Lista apenas as imagens que contêm 'web-app-v1' no nome
docker images | grep web-app-v1
```
2.  **Definir Variáveis:** Configure as variáveis de ambiente com suas informações da AWS.
```bash
# Substitua [SEU_ID_DA_CONTA] pelo seu ID real (12 dígitos)
AWS_ACCOUNT_ID="123123123123" 
AWS_REGION="us-east-2" # Altere se necessário
REPO_NAME="app-frontend"
```

## Seção 2: Criação do Repositório ECR

### 2.1.  **Criação:** Use o AWS CLI para criar o repositório ECR. Se ele já existir, este comando não fará nada.
```bash
aws ecr create-repository \
--repository-name $REPO_NAME \
--region $AWS_REGION
```

### 2.2.  **Obter URI do Repositório:** O URI é o endereço completo onde a imagem será armazenada.
```bash
REPO_URI="$AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com/$REPO_NAME"
echo "URI do ECR: $REPO_URI"
```

## Seção 3: Autenticação (Login Docker)
O cliente Docker precisa de um token de autorização da AWS para se comunicar com o ECR.

### 3.1.  **Obter Token de Login:** Use o `aws ecr get-login-password` para gerar uma senha temporária e passe-a para o `docker login`.
```bash
aws ecr get-login-password --region $AWS_REGION | \
docker login --username AWS --password-stdin $AWS_ACCOUNT_ID.dkr.ecr.$AWS_REGION.amazonaws.com
```
> [!NOTE] 
> **Resultado Esperado:** Mensagem: "Login Succeeded".

## Seção 4: Tagging e Push (Upload)
Esta é a fase final, que garante que o Docker saiba para onde enviar a imagem.

### 4.1.  **Tagging da Imagem:** Marque sua imagem local (`web-app-v1`) com o URI completo do ECR.
```bash
docker tag web-app-v1:$IMAGE_TAG $REPO_URI:$IMAGE_TAG
```
> [!NOTE] 
> **Verificação:** `docker images | grep $REPO_URI` deve mostrar a imagem com o novo nome.

### 4.2.  **Push Final:** Envie a imagem marcada para o ECR.
```bash
docker push $REPO_URI:$IMAGE_TAG
```
> [!NOTE] 
> Observe o progresso do *upload* de cada *layer*.

## Seção 5: Verificação Remota 
### 5.1.  **Verificação ECR:** Use o AWS CLI para confirmar que a imagem foi carregada no repositório.
```bash
aws ecr describe-images \
--repository-name $REPO_NAME \
--region $AWS_REGION \
--query 'imageDetails[].imageTags[0]'
```
> [!NOTE] 
>**Resultado Esperado:** A tag `v1.0` (ou a sua tag) deve ser exibida, comprovando o *upload*.

# Seção 6: BONUS Usando a Imagem do ECR no EKS
### 6.1. Definir nome do cluster
```bash
CLUSTER_NAME="cluster-eks-unifaat"
```
### 6.2. Criar uma role IAM para o cluster EKS
```bash
# Criar role
aws iam create-role \
  --role-name EKSClusterRole \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [
      {
        "Effect": "Allow",
        "Principal": {
          "Service": "eks.amazonaws.com"
        },
        "Action": "sts:AssumeRole"
      }
    ]
  }'
```
### 6.3. Anexar política necessária à role
```bash
# Anexar política
aws iam attach-role-policy \
  --policy-arn arn:aws:iam::aws:policy/AmazonEKSClusterPolicy \
  --role-name EKSClusterRole
```

### 6.4. Criar role IAM para os worker nodes
```bash
aws iam create-role \
  --role-name EKSNodeRole \
  --assume-role-policy-document '{
    "Version": "2012-10-17",
    "Statement": [{
      "Effect": "Allow",
      "Principal": {"Service": "ec2.amazonaws.com"},
      "Action": "sts:AssumeRole"
    }]
  }'
```
### 6.5. Anexar políticas necessárias para os nodes
```bash
aws iam attach-role-policy --role-name EKSNodeRole --policy-arn arn:aws:iam::aws:policy/AmazonEKSWorkerNodePolicy

aws iam attach-role-policy --role-name EKSNodeRole --policy-arn arn:aws:iam::aws:policy/AmazonEKS_CNI_Policy

aws iam attach-role-policy --role-name EKSNodeRole --policy-arn arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryReadOnly
```
### 6.6. Criar VPC para o cluster (usando CloudFormation stack)
```bash
aws cloudformation create-stack \
  --stack-name eks-vpc \
  --template-url https://s3.us-west-2.amazonaws.com/amazon-eks/cloudformation/2020-10-29/amazon-eks-vpc-private-subnets.yaml
```

### 6.7. Aguardar criação da VPC
```bash
aws cloudformation wait stack-create-complete --stack-name eks-vpc
```
### 6.8. Obter IDs das subnets criadas
```bash
SUBNET_IDS=$(aws cloudformation describe-stacks \
  --stack-name eks-vpc \
  --query 'Stacks[0].Outputs[?OutputKey==`SubnetIds`].OutputValue' \
  --output text)
```
### 6.9. Criar cluster EKS
```bash
# Criar cluster
aws eks create-cluster \
  --name $CLUSTER_NAME \
  --role-arn $(aws iam get-role --role-name EKSClusterRole --query 'Role.Arn' --output text) \
  --resources-vpc-config subnetIds=$SUBNET_IDS \
  --kubernetes-version 1.34
```
### 6.10. Aguardar cluster ficar ativo
```bash
aws eks wait cluster-active --name $CLUSTER_NAME
```

### 6.11. Criar Node Group (ESSENCIAL - sem isso os pods não rodam)
```bash
aws eks create-nodegroup \
  --cluster-name $CLUSTER_NAME \
  --nodegroup-name app-frontend-nodes \
  --node-role $(aws iam get-role --role-name EKSNodeRole --query 'Role.Arn' --output text) \
  --subnets $(echo $SUBNET_IDS | tr ',' ' ') \
  --instance-types t3.medium \
  --scaling-config minSize=2,maxSize=3,desiredSize=2
```
### 6.12. Aguardar node group ficar ativo
```bash
aws eks wait nodegroup-active --cluster-name $CLUSTER_NAME --nodegroup-name app-frontend-nodes
```

### 6.13. Verificar status do cluster
```bash
aws eks describe-cluster --name $CLUSTER_NAME --query 'cluster.status'
```
### 6.14. Configurar kubeconfig
```bash
# Configurar kubeconfig
aws eks update-kubeconfig --name $CLUSTER_NAME
```

### 6.15 Verificar conexão com o cluster
```bash
kubectl cluster-info
kubectl get nodes
```

## Seção 7: Deploy da Aplicação no EKS
### 7.1. Criar um namespace para o app
```bash
kubectl create namespace app-frontend
```

### 7.2. Criar arquivo deployment.yaml com substituição de variáveis
```bash
cat << EOF > deployment.yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: app-frontend
  namespace: app-frontend
spec:
  replicas: 2
  selector:
    matchLabels:
      app: app-frontend
  template:
    metadata:
      labels:
        app: app-frontend
    spec:
      containers:
      - name: app-frontend
        image: ${REPO_URI}:${IMAGE_TAG}
        ports:
        - containerPort: 80
EOF
```

### 7.3. Criar arquivo service.yaml
```bash
cat << EOF > service.yaml
apiVersion: v1
kind: Service
metadata:
  name: app-frontend-service
  namespace: app-frontend
spec:
  type: LoadBalancer
  ports:
  - port: 80
    targetPort: 80
  selector:
    app: app-frontend
EOF
```

### 7.4. Aplicar os manifestos
```bash
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml
```
### 7.5. Verificar status dos recursos
```bash
kubectl get deployments -n app-frontend
kubectl get pods -n app-frontend
kubectl get svc -n app-frontend
```
### 7.6. Aguardar LoadBalancer ficar disponível (pode levar alguns minutos)
```bash
kubectl get svc app-frontend-service -n app-frontend -w
```

### 7.7. Obter o DNS do LoadBalancer
```bash
ENDPOINT=$(kubectl get svc app-frontend-service -n app-frontend -o jsonpath='{.status.loadBalancer.ingress[0].hostname}')
echo "Endpoint: $ENDPOINT"
```
### 7.8. Testar a conexão (aguarde alguns minutos para o LoadBalancer propagar)
```bash
curl http://$ENDPOINT
```

### 7.9. Debug se não funcionar
```bash
# Ver logs dos pods
kubectl logs -n app-frontend -l app=app-frontend

# Descrever deployment
kubectl describe deployment app-frontend -n app-frontend

# Ver eventos
kubectl get events -n app-frontend --sort-by='.lastTimestamp'
```

## Seção 8: Limpeza do Ambiente
> [!IMPORTANT] 
> **⚠️ ATENÇÃO:** A limpeza completa remove todos os recursos criados neste laboratório. Certifique-se de que não precisa mais desses recursos antes de executar os comandos de exclusão.
>
> **⚠️ Importante:** Não esqueça de limpar o ambiente após o uso. Recursos AWS como EKS, CloudFormation, ECR e Load Balancer podem gerar cobranças contínuas enquanto estiverem ativos.

### 8.1 Limpeza do Kubernetes (EKS)

1. **Remover Serviço e Deployment:**
```bash
kubectl delete -f service.yaml
kubectl delete -f deployment.yaml
kubectl delete namespace app-frontend
```

2. **Deletar Node Group (IMPORTANTE - deletar antes do cluster):**
```bash
aws eks delete-nodegroup --cluster-name $CLUSTER_NAME --nodegroup-name app-frontend-nodes
aws eks wait nodegroup-deleted --cluster-name $CLUSTER_NAME --nodegroup-name app-frontend-nodes
```

3. **Deletar Cluster EKS:**
```bash
# Deletar cluster
aws eks delete-cluster --name $CLUSTER_NAME

# Aguardar exclusão
aws eks wait cluster-deleted --name $CLUSTER_NAME
```

4. **Remover VPC e Stack CloudFormation:**
```bash
# Deletar stack da VPC
aws cloudformation delete-stack --stack-name eks-vpc

# Aguardar exclusão
aws cloudformation wait stack-delete-complete --stack-name eks-vpc
```

5. **Remover Roles IAM:**
```bash
# Desanexar políticas do EKSClusterRole
aws iam detach-role-policy \
  --role-name EKSClusterRole \
  --policy-arn arn:aws:iam::aws:policy/AmazonEKSClusterPolicy

# Deletar EKSClusterRole
aws iam delete-role --role-name EKSClusterRole

# Desanexar políticas do EKSNodeRole
aws iam detach-role-policy --role-name EKSNodeRole --policy-arn arn:aws:iam::aws:policy/AmazonEKSWorkerNodePolicy
aws iam detach-role-policy --role-name EKSNodeRole --policy-arn arn:aws:iam::aws:policy/AmazonEKS_CNI_Policy
aws iam detach-role-policy --role-name EKSNodeRole --policy-arn arn:aws:iam::aws:policy/AmazonEC2ContainerRegistryReadOnly
# Deletar role
aws iam delete-role --role-name EKSClusterRole
```

### 8.2 Limpeza do ECR

1. **Remover Imagens do Repositório:**
```bash
# Listar imagens
aws ecr list-images --repository-name $REPO_NAME --region $AWS_REGION

# Deletar imagem específica
aws ecr batch-delete-image \
  --repository-name $REPO_NAME \
  --region $AWS_REGION \
  --image-ids imageTag=$IMAGE_TAG
```

2. **Deletar Repositório ECR:**
```bash
# Deletar repositório (force remove todas as imagens)
aws ecr delete-repository \
  --repository-name $REPO_NAME \
  --region $AWS_REGION \
  --force
```

### 8.3 Limpeza Local (Docker)

1. **Remover Imagens Locais:**
```bash
# Remover imagem com tag do ECR
docker rmi $REPO_URI:$IMAGE_TAG

# Remover imagem local
docker rmi web-app-v1:$IMAGE_TAG

# Remover imagem base (se não for mais necessária)
docker rmi busybox:latest
docker rmi nginx:alpine
```

2. **Limpeza Geral do Docker:**
```bash
# Remover containers parados
docker container prune -f

# Remover imagens não utilizadas
docker image prune -a -f

# Remover volumes não utilizados
docker volume prune -f

# Limpeza completa do sistema
docker system prune -a --volumes -f
```

### 8.4 Verificação Final

```bash
# Verificar imagens Docker locais
docker images

# Verificar repositórios ECR
aws ecr describe-repositories --region $AWS_REGION

# Verificar clusters EKS
aws eks list-clusters --region $AWS_REGION

# Verificar stacks CloudFormation
aws cloudformation list-stacks --stack-status-filter CREATE_COMPLETE
```
